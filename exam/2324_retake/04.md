编写一个名为 filterRng 的函数，该函数接受一个函数列表作为输入。给定列表中的每个函数均可不带参数调用，且可能使用 R 的随机数生成器（RNG），例如通过调用 runif()、sample()、rnorm() 或类似函数实现。

filterRng应返回仅包含在执行过程中不使用RNG的函数的列表。该函数可对fs中的函数进行任意次数调用，且可假设除可能使用RNG外不存在其他副作用。特别地，这些函数不调用set.seed()。

结果列表中的函数应按其在输入中的出现顺序排列。

### 核心思路

R 的随机数生成器会改变全局变量 `.Random.seed`。如果调用函数前后 `.Random.seed` 没变，说明函数**没有使用 RNG**

```r
filterRng <- function(fs) {
  checkmate::assert_list(fs, types = "function")
  
  # 判断单个函数是否使用 RNG
  uses_rng <- function(f) {
    # 保存当前 RNG 状态，记录调用前的状态
    old_seed <- .Random.seed
    
    # 调用函数
    f()
    
    # 检查 RNG 状态是否改变
    changed <- !identical(old_seed, .Random.seed)
    
    # 恢复 RNG 状态，<<- 修改全局变量，保证可重复检测
    .Random.seed <<- old_seed
    
    return(changed)
  }
  
  # 处理边界情况，确保 .Random.seed 存在（首次使用 RNG 前可能不存在）
  if (!exists(".Random.seed")) {
    runif(1)
  }
  
  # 过滤出不使用 RNG 的函数
  Filter(function(f) !uses_rng(f), fs)
  # `uses_rng(f)` 返回 `TRUE` → 使用了 RNG → 排除
  # `uses_rng(f)` 返回 `FALSE` → 没用 RNG → 保留u
}
```
* **理解 `.Random.seed`**
```r
# 初始化 RNG
runif(1)

# 查看当前种子
.Random.seed[1:5]
#> [1]  403  624  ...

# 再次使用 RNG
runif(1)

# 种子改变了
.Random.seed[1:5]
#> [1]  403  624  ...  # 数值不同了
```
